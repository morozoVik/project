# Модуль generators

Модуль содержит генераторы для обработки финансовых транзакций и генерации номеров банковских карт.

## Функции модуля

### 1. `filter_by_currency(transactions, currency)`
Фильтрует транзакции по заданной валюте.

**Параметры:**
- `transactions` - список словарей с транзакциями
- `currency` - код валюты для фильтрации (например, "USD")

**Возвращает:**
- Итератор, возвращающий транзакции с указанной валютой

**Пример использования:**

from src.generators import filter_by_currency

## Получаем только USD-транзакции
usd_transactions = filter_by_currency(transactions, "USD")

## Получаем первые 2 USD-транзакции
for _ in range(2):
    print(next(usd_transactions))

# Тестирование проекта

Проект включает комплексные тесты для проверки основных модулей:

### 1. Тесты виджетов (`test_widget.py`)
Проверяет функции для форматирования финансовой информации и дат:
- `mask_account_card()` - маскировка номеров карт и счетов:
  - Корректное форматирование для Visa/MasterCard/Мир
  - Правильная маскировка счетов (последние 4 цифры)
  - Обработка некорректных данных
- `get_date()` - преобразование дат:
  - Конвертация ISO-формата в "ДД.ММ.ГГГГ"
  - Обработка невалидных и пустых дат

### 2. Тесты обработки данных (`test_processing.py`)
Проверяет функции фильтрации и сортировки транзакций:
- `filter_by_state()`:
  - Фильтрация по статусам EXECUTED/PENDING/CANCELED
  - Обработка записей с отсутствующим state
- `sort_by_date()`:
  - Сортировка по возрастанию/убыванию даты
  - Обработка записей без даты

### 3. Тесты масок (`test_masks.py`)
Проверяет функции маскировки финансовых данных:
- `get_mask_card_number()`:
  - Форматирование 16-значных номеров карт
  - Валидация длины и содержимого номера
- `get_mask_account()`:
  - Маскировка 10-значных номеров счетов
  - Проверка минимальной длины и цифрового формата

### 4. Тесты генераторов (test_generators.py)
Проверяет работу генераторов финансовых данных:
- `test_filter_by_currency()`:
  - Фильтрация транзакций по валюте USD
  - Проверка корректности фильтрации RUB-транзакций
  - Обработка пустого списка транзакций
  - Проверка отсутствия ошибок при несуществующей валюте
- `test_transaction_descriptions()`:
  - Получение описаний выполненных транзакций
  - Проверка порядка вывода описаний
  - Обработка списка без транзакций
- `test_card_number_generator()`
  - Генерация номеров карт в диапазоне 1-5:
     - Проверка формата "XXXX XXXX XXXX XXXX"
     - Валидация последовательности номеров
     - Проверка заполнения ведущими нулями
  - Генерация крайних значений:
     - Номер 0000 0000 0000 0001
     - Номер 9999 9999 9999 9999
  - Проверка обработки недопустимых диапазонов:
     - Стартовое значение больше конечного
     - Отрицательные значения
