# Модуль generators

Модуль содержит генераторы для обработки финансовых транзакций и генерации номеров банковских карт.

## Функции модуля

### 1. `filter_by_currency(transactions, currency)`
Фильтрует транзакции по заданной валюте.

**Параметры:**
- `transactions` - список словарей с транзакциями
- `currency` - код валюты для фильтрации (например, "USD")

**Возвращает:**
- Итератор, возвращающий транзакции с указанной валютой

**Пример использования:**

from src.generators import filter_by_currency

## Получаем только USD-транзакции
usd_transactions = filter_by_currency(transactions, "USD")

## Получаем первые 2 USD-транзакции
for _ in range(2):
    print(next(usd_transactions))

# Декоратор `log`

Декоратор для логирования выполнения функций с возможностью вывода как в консоль, так и в файл.

### Особенности:
- Автоматическое логирование начала и конца выполнения функции
- Запись результатов успешного выполнения
- Фиксация ошибок с указанием типа исключения и входных параметров
- Гибкий вывод: в консоль или файл (по выбору)
- Сохранение метаданных оригинальной функции

### Использование:

from src.decorators import log

## Логирование в консоль
@log()
def example_console(a, b):
    return a + b

## Логирование в файл
@log(filename="app.log")
def example_file(x, y):
    return x * y


# Модуль external_api.py
- `convert_to_rub(transaction: Dict) -> float`

Конвертирует сумму транзакции в рубли по текущему курсу через внешнее API.

### Параметры:

- transaction - словарь с данными транзакции, должен содержать:

- amount (обязательный): сумма (число или строка)

- currency (опциональный): код валюты (3 буквы, по умолчанию 'RUB')

### Возвращает:

- Сумму в рублях (float), округленную до 2х знаков

### Исключения:

- ValueError: если отсутствует сумма, неверный формат валюты или ошибка API

- requests.exceptions.RequestException: при проблемах с сетевым запросом

# Модуль utils.py
- `get_transactions(file_path: str) -> List[Dict]`
- 
Читает JSON-файл и возвращает список транзакций.

### Параметры:

-file_path - путь к JSON-файлу с транзакциями

### Возвращает:

- Список словарей с транзакциями или пустой список, если:

  - файл не найден

  - файл пустой

  - содержимое не является валидным JSON

  - JSON не является списком

# Тестирование проекта

Проект включает комплексные тесты для проверки основных модулей:

### 1. Тесты виджетов (`test_widget.py`)
Проверяет функции для форматирования финансовой информации и дат:
- `mask_account_card()` - маскировка номеров карт и счетов:
  - Корректное форматирование для Visa/MasterCard/Мир
  - Правильная маскировка счетов (последние 4 цифры)
  - Обработка некорректных данных
- `get_date()` - преобразование дат:
  - Конвертация ISO-формата в "ДД.ММ.ГГГГ"
  - Обработка невалидных и пустых дат

### 2. Тесты обработки данных (`test_processing.py`)
Проверяет функции фильтрации и сортировки транзакций:
- `filter_by_state()`:
  - Фильтрация по статусам EXECUTED/PENDING/CANCELED
  - Обработка записей с отсутствующим state
- `sort_by_date()`:
  - Сортировка по возрастанию/убыванию даты
  - Обработка записей без даты

### 3. Тесты масок (`test_masks.py`)
Проверяет функции маскировки финансовых данных:
- `get_mask_card_number()`:

- `get_mask_account()`:
  - Маскировка 10-значных номеров счетов

Проверяет работу генераторов финансовых данных:
